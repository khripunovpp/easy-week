# Easy Week — Guidebook (дизайн-система)

Единый язык интерфейса. Все токены живут в `frontend/src/styles.scss` как CSS-переменные,
общие компоненты — там же глобальными классами. Живая версия — экран **`/guide`** в приложении.

## Принципы
- **Тёплый и спокойный.** Off-white фон с лёгким персиковым градиентом, один тёплый акцент.
- **Мягкие поверхности.** Белые карточки с крупным скруглением и мягкой тенью, без жёстких границ.
- **Домейн-акцент — заморозка.** Фишка продукта — заготовки впрок: везде читается «вакуум + заморозка»
  (бейдж ❄️, синий frost-акцент).
- **Mobile-first.** Один столбец, крупные тапы, нижний таб-бар, шапки/футеры с фейд-блюром.

### Тёмная тема
Тёплая тёмная палитра (коралловый акцент сохраняется). Все токены выше переопределяются в
миксине `ew-dark` (`styles.scss`): фон `#201a17→#17130f`, `--surface #2a2320`, текст `--ink
#f2ebe6`, акцент `--accent #f07a58`, тени — на чёрном. Переключение — атрибут `data-theme` на
`<html>` (ставит `PreferencesService`): `:root[data-theme="dark"]` — ручная тёмная,
`[data-theme="light"]` — ручная светлая, без атрибута — «Система» (тёмная под
`@media (prefers-color-scheme: dark)`). Цвет статус-бара (`<meta name="theme-color">`) сервис
синхронизирует: светлая `#fbe9e1`, тёмная `#201a17`. **Новый UI не хардкодит цвета — только
токены, тогда тёмная тема работает автоматически.**

## Типографика
- **Display** (`--font-display`, Bricolage Grotesque) — заголовки `h1–h3`, вес 700, letter-spacing −0.01em.
- **Body** (`--font-body`, Inter) — текст и UI. База 16px / 1.45.
- Размеры: h1 27px · h2 20px · заголовок карточки 16px/700 · текст 15px · подписи 13px · eyebrow 12px.
- **iOS-правило:** у любого `input` — `font-size: 16px` (иначе Safari зумит при фокусе).

## Ритм и форма
- Отступы: `--space-1..5` = 6 / 10 / 16 / 22 / 30 px.
- Радиусы: `--r-card` 28 · `--r-md` 18 · `--r-badge` 16 (эмодзи-бейджи) · `--r-sm` 12 · `--r-pill` 999.
- Тени: `--shadow-soft` (пилюли/строки) · `--shadow-card` (карточки) · `--shadow-float` (меню/поповеры).

## Компоненты (глобальные классы)

- **Кнопки** — `.btn-primary` (коралловая пилюля) и `.btn-ghost` (утопленная). Полная ширина, `--r-pill`,
  жмётся на `:active`.
- **Теги статуса** — `.tag` + модификаторы `.tag--ok` (принят, зелёный) / `.tag--no` (отклонён, красный) /
  `.tag--draft` (черновик, коралловый).
- **Лоадер** — `ew-cooking` (кастрюлька с паром + смена ~100 кулинарных фраз). Единый индикатор
  загрузки/генерации во всём приложении; спиннеров нет.
- **Карточка** — `.card` (белый фон, `--r-card`, `--shadow-card`).
- **Круглый чекбокс** — контур `--line`, отмеченный — заливка `--done` с белой галочкой + зачёркивание.
- **Эмодзи-бейдж блюда** — `.emoji-badge` (50px, `--r-badge`) + пастель `.pastel-0..4` по индексу.
- **Мета-чип** — `.meta-chip` (время/порции, `--ink-2`, иконка 14px).
- **Бейдж заморозки** — `.badge-frost` (`❄️ N дн`, `--frost` на `--frost-soft`).
- **Нижний таб-бар** — 5 вкладок, полупрозрачный白, активная — коралловая.
- **Выпадашка** — пилюля-триггер + карточка `--shadow-float`, активный пункт на `--accent-soft`.

### Лейаут страницы (`.page`)
Единый каркас для всех экранов — общие классы в `styles.scss`, страница не считает
отступы сама.

```html
<div class="page page--header-fixed page--footer-fixed">
  <header class="page__header page__header--fixed fade-top">
    <a class="icon-btn page__back">…</a>          <!-- опционально: назад -->
    <div class="page__heading">                    <!-- опционально: тайтл + сабтайтл -->
      <h1 class="page__title">Покупки</h1>
      <p class="page__subtitle">Куриный максимум</p>
    </div>
    <div class="page__actions">…</div>             <!-- контролы/кнопки/выпадашка справа -->
  </header>

  <div class="page__inner">…контент (скролл)…</div>

  <footer class="page__footer page__footer--fixed fade-bottom">
    <button class="btn-primary">…</button>
  </footer>
</div>
```

- **`.page`** — flex-колонка на всю область (host компонента прозрачен: `:host { display: contents }`).
- **`.page__header`** / **`.page__footer`** — шапка/футер. Модификатор `--fixed` выносит их
  из потока поверх контента (фейд-блюр вешается классами `.fade-top`/`.fade-bottom`). Без `--fixed`
  панель в потоке — **визуально отступы те же**, за это отвечают модификаторы страницы
  `.page--header-fixed` / `.page--footer-fixed` (задают `page__inner` сдвиг = высота панели с паддингом).
- **`.page__inner`** — скролл-контейнер, единый горизонтальный ритм (`--page-pad-x`).
- **Шапка** может содержать: `.page__back` (слева), `.page__heading` (тайтл + опц. сабтайтл,
  фикс-высота — поздний async-сабтайтл не двигает тайтл), `.page__actions` (справа: кнопки,
  иконки, выпадашка, ссылки).
- **`.icon-btn`** — круглая иконка-кнопка единой высоты `--ctl-h` (назад, экспорт, новый чат…).
- Кастомный футер (напр. композер чата) можно позиционировать только через `.page__footer--fixed`
  и задать `page__inner { padding-bottom }` вручную.

**Токены** (`styles.scss`): `--page-pad-x` (гориз. ритм), `--page-pad-y` (паддинг панелей),
`--page-header-h`/`--page-footer-h` (высота **контента** панели, без паддинга),
`--page-footer-pad-b` (зазор футера до таб-бара, 12px), `--ctl-h` (высота контролов).
Сдвиг контента считается как `контент + паддинг` — не задваивать зазор.

Используется в: Главная, Чат, Планы, План, Покупки, Печать, Блюдо.

### Фейд-блюр шапок и футеров
Плавный переход «контент → панель» вместо жёсткого обреза. Реализовано глобальными классами:

- **`.fade-top`** — липкая шапка сверху. Псевдоэлемент `::before` с `backdrop-filter: blur` и
  градиентной маской (плотно у верха, прозрачно снизу): контент уходит под шапку, размываясь.
- **`.fade-bottom`** — футер/композер снизу, зеркально (прозрачно сверху → плотно снизу).
- Прямые дети `.fade-top/.fade-bottom` автоматически поднимаются над блюр-слоем
  (`> * { position: relative; z-index: 1 }`) — иначе Safari прячет контент за `backdrop-filter`.
- Скролл-контейнер получает `padding-top`/`padding-bottom` под высоту панели.

**Как применять:** вешается на `.page__header--fixed` / `.page__footer--fixed` (см. «Лейаут страницы»);
позиционирование, блюр и маска — из глобального класса.

### Отступ до навигации
Панель-футер (`.fade-bottom`) имеет нижний паддинг **12px** — единый зазор между кнопкой/полем и таб-баром
на всех экранах.

## Движение
- Нажатие: `transform: scale(.98)` + смена фона, 120–150ms.
- Лоадер `ew-cooking`: пар всходит (стаггер), фразы меняются каждые 1.7s.
- Меню/появления: короткие 140–160ms.
- Уважаем `prefers-reduced-motion` (анимации гасятся глобально).

## Правила
1. Никаких хардкод-цветов/радиусов в компонентах — только токены.
2. Общий паттерн (кнопка, тег, спиннер, фейд-панель) — из глобального класса, не дублировать в компоненте.
3. Страница строится на каркасе `.page` (шапка / `.page__inner` / футер) — свой лейаут и отступы
   не изобретать; горизонтальный ритм один (`--page-pad-x`), контролы шапки одной высоты (`--ctl-h`).
4. Домейн-акцент (заморозка/вакуум) виден на карточке блюда и странице рецепта.
5. Всё проверяем на мобильном вьюпорте; `input` — 16px.

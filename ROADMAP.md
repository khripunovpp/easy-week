## ✅ Сделано
- **Варианты рецепта по моделям** — блюдо хранит по варианту рецепта на каждую модель
  (deepseek/gemini/claude/cloudflare) с переключением; бейдж «🤖 модель1 | модель2 | ⟳» под
  названием, ⟳ — попап выбора ещё не сгенеренной модели, одной моделью повторно не генерим.
  Смена активного варианта авто-пересобирает список покупок. Закрыло «generate_details → лучшая модель».
- **Доработка плана в чате (базовая)** — добавить/убрать/заменить блюдо и пересобрать меню
  по просьбе (tool calling / structured actions), правка создаёт новую версию плана.
- **Правка плана → новая карточка внизу чата** — после реплики «Готово…» новая версия плана
  уезжает вниз ленты, прошлая сворачивается как отклонённая, автоскролл к новой.
- **Отклонённый план в чате** — компактно: только название, статус и ссылка «Посмотреть план».
- **Модель чата = модель последнего плана** — при «Продолжить обсуждение» модель чата берётся
  из провайдера последнего плана (а не глобального дефолта профиля).
- **Markdown в репликах бота** — ответы моделей рендерятся из markdown (жирный/списки/код/ссылки).
- **Кнопка «вниз» в чате** — когда лента прокручена не до конца.
- **PDF одной кнопкой + шэр файлом** — «⬆ PDF»; по HTTPS уходит сам файл (Web Share), по LAN-HTTP
  открывается настоящий PDF сервера (без blob:-ссылки, которую iOS слал вместо файла).
- **Ленивая генерация деталей (экономия токенов)** — план отдаёт только «шапки» блюд;
  ингредиенты+шаги+советы генерятся DeepSeek при открытии блюда (и добэкфиллятся для покупок/PDF),
  кэшируются. Output плана упал ~70%. Закрыло правило «generate_details → DeepSeek».
- **Мониторинг** (Prometheus + Grafana + Loki + Promtail): `/metrics` бэкенда
  (токены, кэш, HTTP), логи (journald + AI-JSONL) в Loki, дашборд в Grafana.
  Локально — Docker (`docker/`, профиль `monitoring`), на Пае — native (`install-pi.sh`).
- **Полное логирование AI** — консоль + файл-за-день JSONL (промпт/ответ/usage/кэш).
- **Удаление планов** с подтверждением (модалка, необратимо) + `.btn-danger`.
- **Общее время готовки** плана — на вью плана и в списке.
- **Отмеченные покупки** уезжают вниз своей категории.
- **Убраны спиннеры** — единый кулинарный лоадер `ew-cooking` везде.
- **Единый лейаут по гайдбуку** — все экраны (Главная, Чат, Планы, План, Покупки, Печать, Блюдо)
  на каркасе `.page`: единый ритм, шапки одной высоты, фикс/не-фикс совпадают. Раздел в GUIDEBOOK.md.
- **PDF плана** — бэкенд собирает PDF (fpdf2 + DejaVu, надёжная кириллица); тумблеры
  Рецепты/Покупки с чекбоксами на странице печати. (Шэр файлом — см. пункт выше.)
- **Секции на «Планах»** — Принятые (последние 3 + «Загрузить ещё» +10) и История (отклонённые+черновики).
- **Даты на следующую неделю** — план всегда на ближайшую неделю (готовка на выходных перед ней).
- **Потоковая генерация плана** (SSE): блюда прилетают по одному, лоадер внутри карточки,
  автоскролл, кнопки только по готовности. Тот же вызов DeepSeek — токенов не больше.
- **Потоковый список покупок** (SSE) из детерминированной базы — мгновенно и точно
  (модель в потоке теряла числа, поэтому убрана из этого пути).
- **Деплой на Pi** (192.168.1.230:8080) + репозиторий на GitHub (khripunovpp/easy-week);
  Пай теперь тянет обновления через git (deploy/update.sh).
- Кулинарный лоадер (кастрюлька с паром) + пул ~100 фраз вместо спиннеров.
- Метки в логах: какой запрос уходит в какой API (провайдер · модель · назначение).
- DeepSeek для плана + короткие шаги; развёрнутые инструкции и список — Cloudflare.
- Список покупок в граммах у источника (без хаоса единиц).
- Локальная разработка в Docker с живыми логами; SW погашен в dev-сборке.
- Экран приветствия, выбор кол-ва блюд, фейд-блюр шапок/футеров, вью плана.

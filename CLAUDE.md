# Easy Week — правила проекта

## Роутинг AI-моделей (главное правило)

**Придумывание рецептов — только лучшей моделью.** Сейчас это **DeepSeek**
(`deepseek-chat`). Сюда относится генерация плана (блюда + короткие шаги) и
развёрнутые пошаговые рецепты. Если появится модель лучше — рецепты переезжают на неё.

**Все остальные действия — лучшей бесплатной моделью Cloudflare Workers AI.**
Сейчас это mistral (`@cf/mistralai/mistral-small-3.1-24b-instruct`): нормализация/
валидация, вспомогательные задачи. (Список покупок сейчас собирается детерминированно
из ингредиентов — без модели.)

Провайдеры логируются с меткой: `AI → DeepSeek · …` / `AI → Cloudflare · … · назначение`.

### Текущее соответствие правилу
- `generate_plan` (план + короткие шаги) → DeepSeek ✅
- `generate_details` (развёрнутый рецепт по клику) → сейчас mistral ⚠️ **надо перевести на DeepSeek**
  (это тоже придумывание рецепта). Фолбэк-пайплайн плана на Cloudflare — только когда DeepSeek недоступен.

## Дизайн (обязательно)

**Перед любой задачей по вёрстке/UI — сверяйся с `GUIDEBOOK.md`.** Это источник правды по
дизайн-системе: токены, типографика, лейаут-каркас `.page`, глобальные компоненты, правила.
Новый UI строим на существующих классах/токенах, не изобретаем свой лейаут и не хардкодим
цвета/отступы. Если гайдбук чего-то не покрывает — сначала дополняем гайдбук, потом код.

## Прочее
- Бэклог и хотелки — в `ROADMAP.md`.
- Единицы ингредиентов задаём у источника: только `г` / `мл`, `шт` — редко (штучное).
